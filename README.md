# 代码审核与解释系统

## 1. 系统概述

本系统是一个基于大模型的代码审核、评分和解释系统，旨在帮助开发者提高代码质量和理解代码逻辑。系统支持多种大模型服务，包括 OpenAI、本地部署模型、Azure OpenAI 和 Anthropic。

## 2. 核心功能

### 2.1 代码审核
- **全面的代码分析**：对代码进行质量评估、潜在 bug 检测、代码风格检查、安全性分析和性能优化建议
- **评分系统**：为代码提供 1-10 分的总体评分
- **多语言支持**：支持 JavaScript、Python、Java、C、C++、C#、Go、Rust、PHP、Ruby 等多种编程语言
- **多种输入方式**：支持上传单个代码文件、整个项目目录或直接输入代码内容

### 2.2 代码解释
- **详细的代码说明**：解释代码的功能、逻辑流程、关键算法和技术点
- **清晰易懂**：确保解释内容清晰明了，便于理解
- **多语言支持**：与代码审核相同的语言支持范围
- **项目工程分析**：支持对整个项目目录进行分析和解释

### 2.3 模型配置
- **多种模型选择**：支持 OpenAI、本地部署模型、Azure OpenAI 和 Anthropic
- **详细的配置选项**：为每种模型类型提供相应的配置参数
- **配置持久化**：配置保存到本地存储，下次访问时自动加载
- **模型测试**：提供连接测试功能（开发中）

## 3. 技术实现

### 3.1 前端
- **技术栈**：HTML、CSS、JavaScript
- **界面设计**：响应式布局，用户友好的交互界面
- **数据存储**：使用浏览器本地存储保存模型配置
- **文件上传**：支持代码文件上传功能

### 3.2 后端
- **技术栈**：Node.js、Express 框架
- **模型集成**：支持多种大模型 API
- **文件处理**：使用 multer 处理文件上传
- **API 接口**：提供 `/review` 和 `/explain` 两个核心接口

### 3.3 模型支持
- **OpenAI**：支持 GPT-4、GPT-4 Turbo、GPT-3.5 Turbo
- **本地部署模型**：支持任何兼容 OpenAI API 格式的本地模型（如 Llama 3、Mixtral 等）
- **Azure OpenAI**：支持 Azure 部署的 OpenAI 模型
- **Anthropic**：支持 Claude 3 系列模型

## 4. 使用方法

### 4.1 首次使用
1. 访问系统主页：`http://localhost:3000`
2. 点击右上角的 "⚙️ 模型配置" 链接
3. 选择模型类型并填写相应的配置信息
4. 点击 "保存配置" 按钮
5. 返回主页开始使用

### 4.2 代码审核流程
1. 在主页选择编程语言
2. 选择代码输入方式（上传文件或直接输入）
3. 上传代码文件或输入代码内容
4. 点击 "审核代码" 按钮
5. 等待系统分析完成，查看审核结果

### 4.3 代码解释流程
1. 在主页选择编程语言
2. 选择代码输入方式（上传文件或直接输入）
3. 上传代码文件或输入代码内容
4. 点击 "解释代码" 按钮
5. 等待系统分析完成，查看解释结果

## 6. 注意事项

1. **API Key 安全**：API Key 仅存储在浏览器本地，不会上传到服务器
2. **模型选择**：不同模型的性能和成本不同，请根据实际需求选择
3. **代码长度**：过长的代码可能会导致处理时间增加或超出模型上下文限制
4. **本地模型**：使用本地模型需要确保本地服务已启动且可访问
5. **网络连接**：使用云端模型需要稳定的网络连接
